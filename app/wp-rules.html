Правила разработки на Wordpress

Правила

  • Не засорять function php, вставлять функции в отдельный файл из папки inc, файл подключать в function.php через require
  • файл .css если больше 512кб подключать отдельно для каждой страницы, через условия (if else) в function.php
  • Верстка поискаСмотреть
  • Вывод CSS в тему

Создание страниц

  1. создать страницу в пу и задать slug (запомнить slug)
  2. дублировать файл page.php
  3. добавить к названия файла slug страницы через тире (page-slugname.php)
  4. создать файл контента в папке template-parts (content-slugname.php)
  5. в файле page-slugname.php подключить файл контента через функцию get_template_part

Верстка блога на WP

  • при использование старт. шаблона undescores, редактировать файл index.php
  • настраивать контент в template-parts/content.php
  • для сайта-блога в ПУ настройках/чтение оставляем "Ваши последние записи"
  • для других сайтов с блогом в ПУ настройках/чтение ставим статические страницы для главной и для блога
  • страницу блога в папке теме не создаем! Используем index.php для редактирования
  • страницу блога создаем только в ПУ для того чтобы был доступ по ссылке
  • для вывода отдельных категорий блога создаем рубрики и присваем постам (редактировать в archive.php)
  • для создания нескольких блогов нужно создать собственный шаблон подробнее
